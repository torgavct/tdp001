> Började med att komma fram till katalogen dÄr min websida befinner sig i.

cd /TDP001/Blog/

> Inuti i min blog katalog så använda jag kommandot init för att koppla upp till Gitlab.
> Därefter fick jag tillbaks ett meddelande om att jag har initializerad en empty git.

chrto494@CT-Comp:~/TDP001/Blog$ git init

> Sedan utförde jag en git status för att se om filerna har funnits.
> Och fick tillbaks att filerna finns men de har inte addad in i git.

chrto494@CT-Comp:~/TDP001/Blog$ git status

modified: index.html
modified: style.css

> Nu så behövde jag git add filerna in i git och sedan checka git status om filerna har kommit in i git (* = alla filer)

chrto494@CT-Comp:~/TDP001/Blog$ git add *
chrto494@CT-Comp:~/TDP001/Blog$ git status

modified: index.html
modified: style.cs

> Nu med filerna inne i git så kan jag git commit för att skicka det till gitlab.

chrto494@CT-Comp:~/TDP001/Blog$ git commit -am "första commit"
master bc61657] ändringar
2 files changed, 2 insertions(+), 0 deletions(-)

> Men för att skicka över det behövs det använda git push.
> Dessutom så behövs min gitlab username och lösenord
> Och med detta så har det överfört till gitlab

chrto494@CT-Comp:~/TDP001/Blog$ git push
Username for 'https://gitlab.liu.se':
Password for 'https://chrto494@gitlab.liu.se':

> Så nu inne i gitlab så kommer en tab upp med en merge projectet knapp sedan komer den fram längst ner i sdian igen!

> Tillslut det sista steget är att trycka på merge knappen som kommer då klistar in fillerna i gitlab nu permanent
